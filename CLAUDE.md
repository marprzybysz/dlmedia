# dlmedia

TUI media downloader for YouTube and Spotify, built for WSL/Linux.

## Overview

Bash script using `dialog` for TUI interface. Downloads via `yt-dlp` (YouTube) and `spotdl` (Spotify).

## Usage

```bash
./dlmedia              # interactive TUI
./dlmedia --setup      # re-run config wizard
./dlmedia --update     # update yt-dlp
./dlmedia --help       # show help
```

## Dependencies

Required: `dialog`, `yt-dlp`, `ffmpeg`, `python3`
Optional: `spotdl` (for Spotify, install via `pipx install spotdl`)

## Config

`dlmedia.conf` (auto-generated by setup wizard on first run):
- `DOWNLOAD_DIR` ÔÇö download destination path
- `DEFAULT_FORMAT` ÔÇö mp3 or mp4
- `DEFAULT_QUALITY` ÔÇö audio bitrate or video resolution
- `ASK_FORMAT` ÔÇö true/false, prompt user for format each time
- `ASK_QUALITY` ÔÇö true/false, prompt user for quality each time
- `PLAYLIST_SUBFOLDERS` ÔÇö true/false, create subfolders per playlist/album

## Features

- YouTube: single videos, playlists (all/pick/range/single)
- Spotify: albums, playlists via spotdl
- Preview info before download (title, duration, size)
- Overwrite/skip existing files
- Setup wizard on first run (mandatory)
- Summary with error reporting after download
- Return to URL input after download or exit

## Architecture

Single file `dlmedia` script (~400 lines). Config stored in `dlmedia.conf` next to the script. All UI via `dialog` ÔÇö download phase uses `--programbox` for verbose output.

## Control flow

The whole script is one file, read top to bottom:

1. **Defs** ÔÇö `die()`, `check_deps()`, `run_setup()`.
2. **Config load** ÔÇö defaults set as plain vars, then `dlmedia.conf` is `source`d over them (conf overrides defaults). `--update`/`--setup`/`--help` are handled here and exit. No conf Ôćĺ mandatory `run_setup`.
3. **Main `while true` loop** ÔÇö prompt URL Ôćĺ dispatch by URL pattern Ôćĺ download Ôćĺ summary screen with "Powr├│t" (`continue`, back to URL) or "Exit" (`exit 0`).

URL dispatch is string-matching, not validation:
- `*open.spotify.com*` / `*spotify.link*` Ôćĺ Spotify branch (spotdl); it `continue`s at its end so it never falls through to yt-dlp.
- `*list=*` / `*/playlist*` / `*/sets/*` Ôćĺ playlist handling (`/sets/` is SoundCloud-style), then yt-dlp.
- otherwise Ôćĺ single video.

## `dialog` idioms (copy exactly)

- **Result capture:** every interactive `dialog` uses `3>&1 1>&2 2>&3` to get the selection onto stdout into `$(...)`. dialog draws UI to stderr, writes the result to fd 3.
- **Cancel handling:** prompts end with `|| exit 0` (Cancel/Esc) or `|| die "..."`. Never omit this ÔÇö it traps the user.
- **Dynamic sizing:** list dialogs compute height from item count, clamped to `$(tput lines) - 4` (`max_h`/`box_h`/`menu_h`). Reuse for any new scrollable list.
- **Download phase = `--programbox`:** `stdbuf -oL <tool> ... 2>&1 | tee "$log" | dialog --programbox`. `stdbuf -oL` (+ yt-dlp `--newline`) forces line-buffering so output streams live; `tee` keeps a log for the summary. Never use raw `tput` scroll output ÔÇö the user rejected it as "surowe".

## Gotchas (don't reintroduce)

- **Summaries are parsed from the log**, not exit codes. yt-dlp: `grep -c "\[download\] Destination:"` and `"has already been recorded"`, errors via `grep -iE "ERROR:|unable to download"`. spotdl: counts `^Downloaded`, greps `error|failed|skipping`. If you change tool flags/versions, re-verify these patterns match real output.
- **`grep -c` double-zero:** `grep -c` prints "0" *and* exits 1; `|| echo 0` then appends a second "0". Use `|| true` plus an empty-check (`[[ -z ÔÇŽ ]] && x=0`).
- **spotdl stdout leak:** suppress metadata fetch with `>/dev/null 2>&1` (not just `2>/dev/null`) or "Processing query..." bleeds above the dialog.
- **spotdl has no `--list`:** get track metadata via `spotdl save --save-file <json>`, parse with inline `python3`.
- **Subshell vars:** `cmd | while read` loses vars; use `while read < <(cmd)` (process substitution).
- **Overwrite detection scope:** only check the playlist subfolder (via `list_name`/`pl_name`), not the whole `DOWNLOAD_DIR`, or unrelated files trigger the overwrite prompt.
- **Skip-existing differs by tool:** yt-dlp uses `--download-archive "$DOWNLOAD_DIR/.ytdl-archive"`; spotdl uses `--skip-existing`.
- **Custom-path textbox starts empty** (not pre-filled with `$(whoami)`): WSL user `marprzybysz` Ôëá Windows folder `Marcin`, which caused permission-denied.
- **No auto-update:** never update yt-dlp on normal runs; only via `./dlmedia --update`.
- **ASK_FORMAT / ASK_QUALITY must stay:** users need to pick "zawsze pytaj" in setup and toggle these. Removing them is a hard no.
